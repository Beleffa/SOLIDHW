# SOLIDHW
* S (SRP): Логика распределена между 5 классами. GameEngine только управляет процессом, IInputService только читает, IOutputService только пишет, ISettings хранит правила, IIntGenerator считает. Это позволяет менять тексты сообщений или способ генерации числа, не затрагивая саму игру.
* O (OCP): Класс GameEngine закрыт для модификаций, но открыт для расширения через интерфейсы. Например, можно создать другой сервис ввода данных, BotInputService, который будет играть сам, не меняя код движка.
* L (Liskov Substitution): 
    * RegularSettings проверяет валидность данных в конструкторе, гарантируя, что система не перейдет в некорректное состояние (например, отрицательные попытки). 
    * IInputService возвращает int?, что позволяет корректно обрабатывать ошибки парсинга.
* I (Interface Segregation): Интерфейс IIntGenerator отделен от настроек (settings) всей игры. Он принимает только два параметра int, что делает его независимым от количества попыток или других будущих настроек.
* D (Dependency Inversion): GameEngine зависит только от интерфейсов. Все зависимости внедряются через конструктор, что позволяет легко подменять их.
